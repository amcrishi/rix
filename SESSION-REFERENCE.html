🚀Deployment Flow

  1. Push to GitHub → git add -A && git commit -m "message" && git push
  2. Vercel auto-deploys frontend from frontend/ directory
  3. Render auto-deploys backend using render-build.sh
  4. Database migrations run automatically during Render build (prisma migrate deploy)

⚠️ Free Tier Limitations

Render: Backend sleeps after 15 min of inactivity. First request after sleep takes ~30-60 seconds (cold start).

Neon: Database auto-suspends after 5 min of inactivity. Reconnects in ~1 second.

Vercel: Never sleeps. Always fast. Generous free tier.

🛠️Common Commands

Local Development

# Install all dependencies
cd backend && npm install
cd ../ai-engine && npm install
cd ../database && npm install
cd ../frontend && npm install

# Start backend (port 5000)
cd backend && npm run dev

# Start frontend (port 3000)
cd frontend && npm run dev

# Run database migrations
cd database && npx prisma migrate dev

# Open Prisma Studio (GUI for DB)
cd database && npx prisma studio

Deployment

# Push changes (triggers auto-deploy on both Vercel & Render)
git add -A
git commit -m "your message"
git push

# Force redeploy on Render (if auto-deploy doesn't trigger)
# → Render Dashboard → Manual Deploy → Deploy latest commit

# Force redeploy on Vercel
# → Vercel Dashboard → Deployments → ⋮ → Redeploy

Database

# Create a new migration
cd database && npx prisma migrate dev --name your_migration_name

# Apply migrations to production (runs automatically on deploy)
cd database && npx prisma migrate deploy

# Generate Prisma client after schema changes
cd database && npx prisma generate

# Check tables in Neon
# → console.neon.tech → SQL Editor → SELECT tablename FROM pg_tables WHERE schemaname = 'public';

🔧Troubleshooting

Backend not responding

  • Check Render dashboard — is it "Live" (green)?
  • If sleeping, make a request and wait 30-60s for cold start
  • Check logs for errors in Render → your service → Logs tab

Database connection fails

  • Verify DATABASE_URL in Render env vars matches Neon connection string
  • Make sure it ends with ?sslmode=require
  • Check Neon dashboard — is the project active?

Frontend can't reach backend (CORS errors)

  • Ensure CLIENT_URL on Render matches your Vercel domain exactly
  • Ensure BACKEND_URL on Vercel matches your Render URL exactly
  • After changing env vars on Vercel, redeploy manually

Prisma migration issues

  • Never delete migration files from database/prisma/migrations/
  • If stuck, check Neon SQL Editor for _prisma_migrations table
  • For fresh start: drop all tables in Neon, then redeploy on Render
